Condições Aninhadas

São condições que contém mais condições dentro dela, dando mais saídas além da situação binária que apenas uma condição permite

Nesse caso, se a condição 1 for atendida, ele faz o comando do primeiro bloco, se falso, o segundo bloco contém outra condição if, para ser testada também como verdadeira ou falsa.

Estrutura


    if (condição 1){
        true
    }else if (condição 2){
        true
    }else{
        false
    }

Exemplo


    var idade
    idade = 67
    if (idade <16){
    console.log('Não vota')
    }else if(idade < 18 || idade >= 65){
        console.log('Voto opcional')
    }else{
    console.log('Vota')
    }

Legal notar nesse exemplo que apesar da idade para voto opcional ser entre 16 e 18 anos, não precisamos colocar a idade mínima pois, como declaramos no if inicial, APENAS as idades maiores que 16 passarão para o teste da segunda condição. Dessa forma, nunca haverá uma idade 15 ou 8 para ser testada, por exemplo, e podemos deixar nosso código mais enxuto!

Condições Múltiplas

Facilitam a escrita do código quando as condições a serem atendidas não compreendem intervalos, mas sim valores fixos a serem comparados. Dessa forma, podemos usar a expressão switch para tal.

Estrutura


    switch(){
        case a:
        break
        case b:
        break
        ...
        default:
        break
    }

Exemplo


    switch(semaforo){
        case verde:
            console.log('Siga')
        break
        case amarelo:
            console.log('Atenção')
        break
        default:
            console.log('Pare')
        break
    }

Nesse exemplo, temos 3 saídas possíveis para um semáforo: Ou ele está verde, ou amarelo, ou vermelho. Como testamos a condição verde no case 1 e a amarela no case 2, só nos resta a vermelha, que foi colocada na default.

Coloque um freio no seu código! Ou vários!!!

Atenção! O comando switch pede o uso do código break depois de cada case colocado. Isso acontece porque o switch vai analisar o resultado e pular para a execução direta do case adequado, mas não vai parar nele, e vai continuar executando tooooodos os cases que aparecerem depois. O uso do código break serve justamente para, depois da execução do case, parar todo o restante do processo do switch e ir para o restante do código.

Códigos Novos

  • new Date()

    • Podemos usar ela na criação de uma variável, para atribuir propriedades de data e hora

  • getHours()

    • Utilizado em conjunto com o new Date, podemos capturar a hora do sistema que está rodando o código. Como exemplo, podemos ter uma variável data que receba o valor new Date e possa nos fornecer esses dados, enquanto em outra variável vamos chamar a hora com data.getHours()

  • getDay()

    • O getDay funciona de forma semelhante ao getHours, mas informando os dias da semana (domingo, segunda, etc). Vale notar, no entanto, que ele retorna o dia da semana em formato numérico, sendo 0 para domingo, 1 para segunda, e assim por diante, até o 6 para o sábado.
