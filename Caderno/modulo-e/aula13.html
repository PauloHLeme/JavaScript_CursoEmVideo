Repetições (Parte 1)

Repetições com teste lógico no início

Várias tarefas do dia a dia requerem repetições. Executar uma tarefa um número n de vezes.

Ao escrevermos um código, isso também acontece bastante. As vezes precisamos fazer a repetição de uma função várias vezes, e para isso temos justamente as Repetições.

Acontece que nem sempre a repetição é linear. Várias vezes temos condições no caminho, que então farão a repetição acontecer ou não. Assim, a repetição tem condições lógicas para acontecer ou não, em um loop que continua enquanto a condição for atendida!

Estrutura de código


    while(condição){

    }

Exemplo


    var c = 1
    while (c <= 50){
        console.log(`Passo ${c}`)
        c++
    }

Esse código vai numerar cada vez que ele escrever, até completar as 50 vezes pedidas pela condição de repetição. Caso tívessemos que fazer isso manualmente, teríamos condição, mas seria um trabalho totalmente desnecessário!

Repetições com teste lógico no final

O funcionamento é similar a repetição com teste inicial, mas ele joga o teste para o final do processo, executanto a tarefa primeiro, e testando a condição no final.

Estrutura de código


    do{

    }while(condição)

Exemplo


    var c = 1
    do{
        console.log(`Passo ${c}`)
        c++ //c recebe c+1
    }while (c <= 6)

Os resultados são muito semelhantes na maioria dos casos, mas podemos encontrar algumas situações onde a existência do teste antes ou depois pode ser bem relevante. No dia a dia, certamente você vai encontrar uma situação onde isso faz a diferença!
